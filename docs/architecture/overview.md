# Minecrust 架构总览

## 简介
Minecrust 是一个基于 Rust 语言编写的 Minecraft 1.21.1 的非盈利性本地复刻版。它使用原版游戏资产，但引擎部分完全从零开始自研构建。

## 核心支柱
1. **纯 wgpu 渲染引擎**: 本项目不使用 Bevy 或其他高级游戏引擎。渲染管线完全基于自定义的 `wgpu` 后端，以提供对体素渲染流程（如计算着色器剔除、贪婪网格化）的最大控制权。
2. **独立的 ECS 架构**: 使用轻量级的实体组件系统（ECS，如 `hecs` 或 `flecs`，亦或是完全自研）来解耦逻辑。
3. **自研 C/S 网络架构**: 从项目初期即支持多人游戏。我们不使用原版的 Minecraft Protocol 767 协议，而是专门针对我们的 ECS 和区块数据结构构建了定制的协议（使用 TCP 或基于 UDP 的 QUIC 协议）。
4. **离线资产处理管线**: 客户端在运行时不直接解析 `1.21.1.jar` 中的 JSON。所有的原版资产将由一个独立的 CLI 工具在离线阶段处理为高度优化的二进制格式（`.mca` - Minecrust Asset），供客户端直接加载。

## 组件划分

### 1. `minecrust-server` (服务端)
权威服务器。
- **世界生成**: 基于 3D 噪声的区块生成。
- **物理模拟**: 权威的物理碰撞与移动检测。
- **状态同步**: 向客户端广播区块数据和实体更新。

### 2. `minecrust-client` (客户端)
专注于渲染和输入的瘦客户端。
- **渲染器**: 基于 `wgpu` 的渲染器，负责处理区块网格、材质图集和基础 UI。
- **网格化 (Mesher)**: 接收服务端发来的区块数据，并在客户端执行贪婪网格（Greedy Meshing）算法。
- **输入与窗口**: 使用 `winit` 处理窗口管理和用户输入。

### 3. `minecrust-asset-cli` (资产处理工具)
离线的资产烘焙器。
- **输入**: 原版的 `1.21.1.jar`。
- **输出**: 优化后的材质图集、方块状态字典以及适用于 `wgpu` 的模型网格数据。

## 项目目录结构 (规划)
```text
minecrust/
├── Cargo.toml (Workspace)
├── docs/                 # 架构文档
├── crates/
│   ├── minecrust-server/ # 独立服务端逻辑
│   ├── minecrust-client/ # wgpu 客户端
│   ├── minecrust-shared/ # 共享结构体 (协议、数学运算、体素数据结构)
│   ├── minecrust-ecs/    # ECS 定义与系统
│   └── minecrust-asset-cli/ # 资产烘焙工具
```
